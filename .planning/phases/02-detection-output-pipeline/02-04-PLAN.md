    def main() -> None:
    Stage 1 — INIT:

    - print(f"[INIT] Audit run started: {today}")

    Stage 2 — AUTH:
    - print("[AUTH] Authenticating to Power BI...")
    - token = get_token()
    - print("[AUTH] Token acquired.")

    Stage 3 — DECISION:
    - snapshot_files = sorted(SNAPSHOTS_DIR.glob("snapshot_*.csv"))
    - if len(snapshot_files) == 0:
        print("[DECISION] No snapshots found — running historical batch fetch (first run).")
        current_df = historical_batch_fetch(token)
      else:
        print(f"[DECISION] {len(snapshot_files)} snapshot(s) found — fetching current is_latest=1 snapshot.")
        current_df = snapshot_fetch(token)

    Stage 4 — BASELINE:
    - print("[BASELINE] Loading snapshots and computing 8-week rolling median baseline...")
    - combined_df = load_snapshots(n=8)
    - combined_df = compute_baseline(combined_df)

    Stage 5 — DETECT:
    - print("[DETECT] Running anomaly detection...")
    - prior_counts = load_prior_counts(today)
    - raw_anomalies = detect_anomalies(current_df, combined_df, threshold_pct)

    - n_increases = (anomalies_df[~anomalies_df["sustained_shift"]]["direction"] == "increase").sum()
    - n_reductions = (anomalies_df[~anomalies_df["sustained_shift"]]["direction"] == "decrease").sum()
    - n_sustained = anomalies_df["sustained_shift"].sum()
    - print(f"[DETECT] Anomalies: {n_increases} increases, {n_reductions} reductions, {n_sustained} sustained shifts (excluded from comment)")

    - print("[NARRATE] Generating Claude narrative...")
    - active_anomalies = anomalies_df[~anomalies_df["sustained_shift"]]

    - print(f"[OUTPUT] Posting ClickUp comment to task {task_id}...")

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env → main() | Secrets (CLICKUP_API_KEY, ANTHROPIC_API_KEY) loaded via python-dotenv; never in audit_config.json |
| anomaly JSON → Anthropic API | Semi-trusted: JSON built from Pydantic-validated DataFrame rows; SKU names from internal PBI data |
| Anthropic API response → ClickUp comment | Output is plain text; no HTML injection risk in ClickUp comment_text field |
| ClickUp API ← post_clickup_comment() | Authorization: raw pk_... token; no Bearer prefix; raise_for_status() gates on error |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Information Disclosure | CLICKUP_API_KEY exposure | mitigate | Key loaded from .env only; .env is git-ignored; audit_config.json has no API key fields; main() raises SystemExit if key is absent rather than proceeding with empty string. |
| T-02-10 | Tampering | ClickUp Authorization header format | mitigate | headers = {"Authorization": api_key} — no "Bearer" prefix. This is enforced by unit test test_config_drives_task_id which verifies the mocked requests.post call. RESEARCH.md Pitfall 1 documents why: pk_... tokens are personal tokens, not OAuth bearer tokens; adding "Bearer" prefix causes 401. |
| T-02-11 | Tampering | BytesIO empty file upload | mitigate | csv_buffer.seek(0) called immediately after to_csv(csv_buffer). RESEARCH.md Pitfall 5 documents this. Enforced by test_csv_attachment_columns verifying the files= parameter structure. |
| T-02-12 | Elevation of Privilege | CLICKUP_TASK_ID misconfigured | mitigate | load_config() raises SystemExit("CLICKUP_TASK_ID is not configured") before any API call if value is "PLACEHOLDER". Human checkpoint at end of Wave 3 requires real task ID before live run. |
| T-02-13 | Tampering | Prompt injection via SKU names in anomaly JSON | accept | SKU names originate from Power BI (OrganiHaus internal data, not external user input). Claude call is one-shot narrative only; no tool use, no code execution, no action taken based on response. Risk is LOW for this internal tool. D-19 already gates that Claude receives pre-computed JSON summary, not raw rows. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan. |
</threat_model>
