## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Power BI API response → pandas | Column names from PBI are trusted but must be normalized before use |
| DAX aggregation → Python join | DAX pre-aggregates to weekly; Python join must not silently drop unmatched keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Column name stripping — `c.split("[")[-1].rstrip("]")` | accept | Pattern is deterministic for PBI-format column names; if PBI changes format, the strip produces unexpected column names that will cause KeyError, surfacing the issue immediately rather than silently |
| T-03-02 | Denial of Service (data integrity) | Left join dropping amzn.gr.* rows | mitigate | `how="left"` in pd.merge preserves all fee_df rows; unjoined count logged to stdout; test_amzn_gr_keys_preserved verifies no rows dropped |
| T-03-03 | Information Disclosure | Currency misassignment (cross-currency averaging in Phase 2) | mitigate | `currency` column derived from `country` per RESEARCH.md Pitfall 6; each row has unambiguous currency label; Phase 2 must group by (key, country), not by (key, sales_region) to avoid mixing EUR currencies |
| T-03-SC | Tampering | pip packages | accept | No new packages installed in this plan; all transformations use pandas 2.2.3 (already verified) |
</threat_model>
